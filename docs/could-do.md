# Could-Do: Future Ideas

Ideas and features worth exploring. Not committed to a roadmap — just things that make sense for the project.

---

## Built-in P2P Communication

Instead of relying on Discord, Slack, or any third-party platform, build community communication directly into the Alexandria app.

### Why

- Aligns with the no-servers, peer-to-peer, offline-first architecture
- Users are already in the app with blockchain-verified identities
- No dependency on any external company or platform
- Community data is owned by the community, not a third party
- Reduces spam and impersonation since identities are already verified
- Dogfoods Alexandria's own P2P and sync infrastructure

### What it could look like

- P2P discussion boards synced the same way course content syncs
- Channels per topic: general, feature requests, bug reports, per-course discussion
- Works offline — messages sync when peers connect
- Tied to existing Alexandria identities (no extra accounts)
- Optional notifications for followed threads

### Interim solution

Until this is built, use GitHub Discussions for structured conversations with the early (developer-heavy) audience. Zero extra infrastructure, already where contributors are.
